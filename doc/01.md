## 第一篇：导入素材并制作为游戏中的精灵组件

### 1. 创建项目

1. 打开 Godot 4.6.2，点击 **“新建项目”**。
2. 项目名称：`ShootGame`
3. 选择一个空文件夹作为项目路径（例如桌面上的 `MyShootGame`）。
4. 渲染器选择 **“Forward+”**（兼容性最好，HTML5 也支持）。
5. 点击 **“创建并编辑”**。

### 2. 获取素材

由于您提供的 `preview.png` 只是一个 Kenney 的标识图，并非游戏实际素材，我们需要下载一套免费的飞行射击游戏图片。

> 推荐使用 Kenney 的 **“Space Shooter”** 素材包（完全免费，公共领域）。
> 下载地址：https://kenney.nl/assets/space-shooter
> 点击下载 `SpaceShooterRedux.zip`

下载后解压，里面会有 `PNG` 文件夹，包含：
- `playerShip1_blue.png`（蓝色玩家飞机）
- `playerShip1_green.png`（绿色玩家飞机）
- `playerShip1_red.png`（红色玩家飞机）
- `enemyShip1.png`、`enemyShip2.png` 等（敌机）
- `laserBlue01.png`、`laserRed01.png` 等（子弹）
- `starfield.png`（星空背景）

如果您想用自己的图片，也可以替换，但本教程将基于这套素材。

### 3. 导入素材到 Godot

1. 在 Godot 编辑器的 **“文件系统”** 面板（右下角），右键点击 `res://` 根目录，选择 **“显示在文件管理器中”**。
2. 将解压后的 `PNG` 文件夹内的 **所有图片文件** 复制到项目文件夹中（例如在项目根目录下新建一个 `assets` 文件夹，再粘贴进去）。
3. 回到 Godot，文件系统会自动刷新，出现 `assets` 文件夹，里面全是 `.png` 文件。

> 注意：如果图片太大，可以在导入设置中调整（但先保持原样，后续可修改）。



### 4. 制作玩家飞机精灵组件

我们将创建一个 **玩家飞机场景**，它是一个可以移动、射击、碰撞的节点组合。

#### 4.1 新建场景
- 点击菜单栏 **场景 → 新建场景**，或者点击左侧 **场景** 面板旁边的 `+` 号。
- 根节点选择 **`CharacterBody2D`**，命名为 `Player`。
- 保存场景：按 `Ctrl + S`，路径 `res://scenes/player.tscn`（需要先新建 `scenes` 文件夹，右键 -> 新建文件夹）。

#### 4.2 添加精灵（视觉）
- 选中 `Player` 节点，点击 **添加子节点** 按钮（或按 `Ctrl + A`），搜索 `Sprite2D`，添加。
- 命名为 `Sprite`。
- 在右侧 **检查器** 中，找到 `Sprite2D` 的 `Texture` 属性，点击 `[空]` → **快速载入** → 选择 `assets/playerShip1_blue.png`（蓝色飞船）。
- 调整大小：由于图片可能较大，在检查器中设置 `Scale` 为 `(0.5, 0.5)`（根据需要调整）。

#### 4.3 添加碰撞体（让飞机能碰撞）
- 选中 `Player` 节点，添加子节点 `CollisionShape2D`。
- 在检查器中，`Shape` 属性点击 `[空]` → **新建 RectangleShape2D**（矩形碰撞体）。
- 调整矩形大小：点击形状旁边的图标（或展开属性），将 `Size` 设置为 `(40, 40)` 左右，使其大致覆盖飞船图片。
- 可以拖动场景中的绿色框来微调。

#### 4.4 添加移动和边界限制的脚本
- 选中 `Player` 节点，点击 **添加脚本** 按钮（文档图标）。
- 脚本路径：`res://scripts/player.gd`（需要先创建 `scripts` 文件夹）。
- 保留 `extends CharacterBody2D`，清空默认内容，写入以下代码：

```gdscript
extends CharacterBody2D

@export var speed := 400.0  # 移动速度

func _physics_process(delta):
    # 获取键盘输入（上下左右/ WASD）
    var input_dir := Input.get_vector("left", "right", "up", "down")
    velocity = input_dir * speed
    
    # 移动
    move_and_slide()
    
    # 限制边界：不出屏幕（根据屏幕大小调整）
    var screen_size = get_viewport_rect().size
    position.x = clamp(position.x, 0, screen_size.x)
    position.y = clamp(position.y, 0, screen_size.y)
```

此时按 `F6`（运行当前场景）可以测试移动（需要先设置输入映射，见下一步）。

#### 4.5 设置输入映射（键盘方向）
- 点击菜单栏 **项目 → 项目设置** → 选择 **输入映射** 选项卡。
- 添加以下动作（每个动作点右下角的“添加”并分配按键）：
  - `left`：A、左箭头
  - `right`：D、右箭头
  - `up`：W、上箭头
  - `down`：S、下箭头
- 添加射击动作 `shoot`：空格键、鼠标左键（后面用）。
- 添加炸弹动作 `bomb`：B 键、Shift 键。

完成后关闭设置。再次运行 `player.tscn`，应该可以用 WASD 或方向键移动蓝色飞船，且不会移出屏幕。

### 5. 制作敌机精灵组件

敌机基本结构与玩家类似，但不需要控制移动方向（会向下自动移动），并且碰撞后玩家会扣血。

#### 5.1 新建敌机场景
- 新建场景，根节点 `CharacterBody2D`，命名为 `Enemy`。
- 保存为 `res://scenes/enemy.tscn`。

#### 5.2 添加精灵和碰撞体
- 添加子节点 `Sprite2D`，纹理选 `assets/enemyShip1.png`（普通敌机），缩放 `(0.5, 0.5)`。
- 添加 `CollisionShape2D`，形状选 `RectangleShape2D`，大小 `(40, 40)` 左右。

#### 5.3 添加脚本（让敌机向下移动）
- 添加脚本 `res://scripts/enemy.gd`，内容：

```gdscript
extends CharacterBody2D

@export var speed := 150.0  # 向下移动速度

func _physics_process(delta):
    velocity = Vector2(0, speed)
    move_and_slide()
    
    # 超出屏幕底部则销毁
    if position.y > get_viewport_rect().size.y + 100:
        queue_free()
```

此时敌机会不断向下移动，超出底部自动删除。

### 6. 制作子弹精灵组件

子弹是玩家和敌机发射的投射物。通常使用 `Area2D` 作为根节点，因为它擅长检测重叠。

#### 6.1 新建子弹场景
- 新建场景，根节点 `Area2D`，命名为 `Bullet`。
- 保存为 `res://scenes/bullet.tscn`。

#### 6.2 添加视觉和碰撞体
- 添加子节点 `Sprite2D`，纹理选 `assets/laserBlue01.png`（蓝色激光），缩放 `(0.5, 0.5)`。
- 添加 `CollisionShape2D`，形状选 `RectangleShape2D`，大小 `(20, 20)` 左右。
- 注意：`Area2D` 需要 `CollisionShape2D` 才能检测碰撞。

#### 6.3 添加移动脚本
- 添加脚本 `res://scripts/bullet.gd`：

```gdscript
extends Area2D

@export var speed := 400.0
var direction := Vector2.UP  # 默认向上

func _physics_process(delta):
    position += direction * speed * delta
    # 超出屏幕销毁
    if position.y < -100 or position.y > get_viewport_rect().size.y + 100:
        queue_free()
```

#### 6.4 区分玩家子弹和敌机子弹
可以通过 `direction` 变量区分：玩家子弹方向向上 `Vector2.UP`，敌机子弹方向向下 `Vector2.DOWN`。后面在发射时设置即可。

### 7. 制作背景（滚动星空）

为了让游戏更有动感，我们添加一个无限滚动的背景。

#### 7.1 新建背景场景
- 新建场景，根节点 `Node2D`，命名为 `Background`。
- 保存为 `res://scenes/background.tscn`。

#### 7.2 添加两个相同的背景图片
- 添加子节点 `TextureRect`（UI 类节点），命名为 `Bg1`。
- 在检查器中，`Texture` 选择 `assets/starfield.png`（如果 Kenney 素材里没有星空，可以用纯黑色图或下载一个星空背景；或者直接用 `Sprite2D` 加一张星空图）。
- 设置 `Stretch Mode` 为 `Cover`，`Size` 设置为屏幕大小（例如 720x1280）。
- 再添加一个 `TextureRect` 命名为 `Bg2`，使用同一纹理。
- 将 `Bg2` 放在 `Bg1` 的正上方（`position.y = -1280`，如果背景高 1280）。

#### 7.3 添加滚动脚本
- 为 `Background` 添加脚本 `res://scripts/background.gd`：

```gdscript
extends Node2D

@export var scroll_speed := 100.0

func _process(delta):
    for bg in [$Bg1, $Bg2]:
        bg.position.y += scroll_speed * delta
        if bg.position.y >= get_viewport_rect().size.y:
            bg.position.y = -get_viewport_rect().size.y
```

这样背景会无限向下滚动，制造飞行向上的错觉。

### 8. 测试所有组件是否正常

目前还没有主游戏场景，但我们可以单独测试每个场景：
- 运行 `player.tscn`：移动测试
- 运行 `enemy.tscn`：它应该向下移动并消失
- 运行 `bullet.tscn`：默认向上移动（需要临时在编辑器里移动一下位置才能看到移动效果）
- 运行 `background.tscn`：背景滚动

## 本篇思考题

**1. 为什么玩家飞机根节点使用 `CharacterBody2D` 而不是 `Area2D`？**  
> 答案：`CharacterBody2D` 专为受外力/速度控制的移动角色设计，内置 `move_and_slide()` 方法可以方便处理碰撞和移动，且不需要自己写复杂的碰撞响应。`Area2D` 更适合检测重叠，但移动时需要手动处理位置和碰撞。

**2. 在设置子弹的碰撞时，为什么 `Area2D` 是合适的根节点？**  
> 答案：子弹通常不需要复杂的物理反应（如反弹、摩擦力），只需要检测到击中敌人或玩家时造成伤害然后消失。`Area2D` 专门用于检测重叠区域，并可以发出信号（`body_entered` 或 `area_entered`），非常适合子弹这种“一碰就没”的投射物。

**3. 如果希望玩家飞机和敌机都有生命值显示，应该在哪个节点上存储生命值变量？**  
> 答案：应该在玩家飞机和敌机各自的根节点（`CharacterBody2D`）上添加脚本并定义 `var health` 变量。这样可以方便地在碰撞时减少生命值，并由各自的脚本负责生命值为零时的死亡逻辑（如播放爆炸特效、增加分数、销毁等）。
