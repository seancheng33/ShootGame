## 第五篇：分数、武器、奖励等内容及测试方法

本篇将完善游戏的数值成长系统，包括：
- **分数系统**：显示当前分数，达到指定分数（64000、128000、256000…）时奖励生命。
- **武器升级系统**：四种武器（常规、激光、霰弹、高威力低速）各自独立等级（1~3级），拾取武器道具后切换并提升该武器等级。
- **奖励道具**：生命道具（+1生命）、炸弹道具（+1炸弹）、武器道具（随机或特定武器）。
- **过关奖励**：击败Boss后奖励一条生命（已在第四篇实现，本篇会确保与分数奖励叠加）。
- **测试方法**：提供调试工具或快捷键快速验证升级和奖励逻辑。

### 第一步：分数UI显示与奖励生命机制

#### 1.1 创建分数显示UI

在 `game.tscn` 场景中添加一个 `CanvasLayer`（确保UI始终在最前），里面放一个 `Label` 显示分数。

1. 打开 `game.tscn`，在根节点 `Game` 下添加子节点 `CanvasLayer`，命名为 `UI`。
2. 在 `UI` 下添加 `Label`，命名为 `ScoreLabel`。设置锚点为右上角（在检查器中点击 Layout → Top Right），调整位置偏移 `(-20, 20)`。
3. 设置字体大小（Theme → Default Font → Size = 30），颜色白色，添加阴影效果更好看。
4. 同样添加一个 `Label` 显示生命值（可选），命名为 `HealthLabel`，放在左上角。
5. 添加一个 `Label` 显示当前武器和等级（可选），命名为 `WeaponLabel`。

#### 1.2 连接分数信号并更新UI

修改 `global.gd`，确保分数变化时发出信号。我们已经有了 `score_changed` 信号。在 `game.gd` 中连接：

```gdscript
# game.gd 的 _ready() 中添加

# 初始化显示
_on_score_changed(Global.score)
```

定义更新函数：

```gdscript
@onready var score_label = $UI/ScoreLabel

func _on_score_changed(new_score):
    score_label.text = "SCORE: " + str(new_score)
    # 检查分数奖励生命
    check_score_reward(new_score)
```

#### 1.3 分数奖励生命逻辑

分数奖励条件：当分数 ≥ 2^n × 1000 且之前未在该阈值奖励过时，奖励一条命。n 从 6 开始（64000），之后每翻一倍奖励一次。

在 `global.gd` 中增加变量记录已奖励的阈值：

```gdscript
var last_reward_threshold := 0  # 记录上次奖励的指数n，初始0表示未奖励任何

func check_score_reward(score):
    var n = 6
    while true:
        var threshold = pow(2, n) * 1000
        if score >= threshold and last_reward_threshold < n:
            # 奖励生命
            var player = get_tree().current_scene.get_node_or_null("Player")
            if player and player.has_method("add_life"):
                player.add_life(1)
                last_reward_threshold = n
                print("Score reward! +1 life at ", threshold)
        elif threshold > score:
            break
        n += 1
```

注意：`last_reward_threshold` 需要在新游戏时重置。在 `reset_game()` 中设置 `last_reward_threshold = 0`。

同时需要在 `player.gd` 中确保 `add_life` 方法不会超过最大生命值。

### 第二步：武器升级系统（独立等级与拾取切换）

#### 2.1 设计武器数据结构

在 `player.gd` 中，我们已经定义了 `WeaponType` 枚举和 `current_weapon`。现在增加每个武器的等级数组：

```gdscript
var weapon_levels := {
    WeaponType.NORMAL: 1,
    WeaponType.LASER: 1,
    WeaponType.SHOTGUN: 1,
    WeaponType.HEAVY: 1,
}
```

并在 `set_weapon` 时更新 `shoot_cooldown` 等属性时，根据当前武器等级调整效果。我们将在 `shoot()` 和 `update_weapon_stats()` 中处理。

#### 2.2 武器等级影响效果

- **常规武器**：等级1单发，等级2双发（平行），等级3三发（扇形）。
- **激光**：等级1单发高伤害，等级2增加穿透能力，等级3增加射速。
- **霰弹**：等级1 3发，等级2 5发，等级3 7发。
- **高威力低速**：等级1单发高伤，等级2子弹变大或爆炸效果，等级3额外弹幕。

为了简化实现，我们仅改变子弹数量或伤害，避免过多复杂逻辑。在 `shoot()` 中根据 `current_weapon` 和 `weapon_levels[current_weapon]` 产生不同的子弹。

重写 `shoot()` 函数（替换之前的简单版本）：

```gdscript
func shoot():
    var level = weapon_levels[current_weapon]
    match current_weapon:
        WeaponType.NORMAL:
            if level == 1:
                fire_bullet(Vector2.UP, 400, 1)
            elif level == 2:
                fire_bullet(Vector2.UP, 400, 1)
                fire_bullet(Vector2.UP + Vector2(0.1, 0), 400, 1)  # 双发
            else:  # level 3
                fire_bullet(Vector2.UP, 400, 1)
                fire_bullet(Vector2.UP.rotated(-0.1), 400, 1)
                fire_bullet(Vector2.UP.rotated(0.1), 400, 1)
        WeaponType.LASER:
            var dmg = 2 + (level - 1)  # 等级2伤害3，等级3伤害4
            fire_bullet(Vector2.UP, 600, dmg)
            if level >= 2:
                fire_bullet(Vector2.UP, 600, dmg)  # 双激光
        WeaponType.SHOTGUN:
            var count = 3 + (level - 1) * 2  # 3,5,7
            var angle_step = 0.15
            var start_angle = -angle_step * (count - 1) / 2
            for i in range(count):
                var angle = start_angle + i * angle_step
                fire_bullet(Vector2.UP.rotated(angle), 400, 1)
        WeaponType.HEAVY:
            var dmg = 3 + (level - 1) * 2  # 3,5,7
            var spd = 200 - (level - 1) * 30  # 越升级子弹速度越慢？但保持低速特性
            fire_bullet(Vector2.UP, max(spd, 140), dmg)
```

同时修改 `fire_bullet` 接受伤害参数，并在子弹实例化时设置 `damage`。

#### 2.3 武器升级与切换方法

添加函数 `upgrade_current_weapon()` 和 `set_weapon_with_level(weapon_type)`：

```gdscript
func upgrade_current_weapon():
    var lv = weapon_levels[current_weapon]
    if lv < 3:
        weapon_levels[current_weapon] = lv + 1
        update_weapon_stats()
        weapon_changed.emit(current_weapon)  # 用于UI更新
        print("Weapon upgraded to level ", weapon_levels[current_weapon])

func set_weapon(weapon_type):
    if current_weapon == weapon_type:
        # 如果拾取同类型武器，则升级
        upgrade_current_weapon()
    else:
        current_weapon = weapon_type
        update_weapon_stats()
        weapon_changed.emit(current_weapon)

func update_weapon_stats():
    # 根据武器类型和等级调整射速等
    var level = weapon_levels[current_weapon]
    match current_weapon:
        WeaponType.NORMAL:
            shoot_cooldown = 0.2 / (0.5 + level * 0.25)  # 等级越高射速越快
        WeaponType.LASER:
            shoot_cooldown = 0.15 / (0.5 + level * 0.2)
        WeaponType.SHOTGUN:
            shoot_cooldown = 0.25 / (0.5 + level * 0.2)
        WeaponType.HEAVY:
            shoot_cooldown = 0.4 / (0.5 + level * 0.15)
```

#### 2.4 武器道具掉落与拾取

在敌机死亡时，有概率掉落道具。我们创建一个道具场景 `powerup.tscn`。

**创建道具场景**：
- 根节点 `Area2D`，命名为 `PowerUp`。
- 添加 `Sprite2D`，根据道具类型使用不同颜色或图标（Kenney素材中有 `powerupRed.png`、`powerupBlue.png` 等）。
- 添加 `CollisionShape2D`，圆形或矩形。
- 添加脚本 `powerup.gd`：

```gdscript
extends Area2D

enum PowerUpType { HEALTH, BOMB, WEAPON_NORMAL, WEAPON_LASER, WEAPON_SHOTGUN, WEAPON_HEAVY }
@export var type := PowerUpType.HEALTH
@export var speed := 100.0  # 下落速度

func _ready():
    area_entered.connect(_on_picked_up)
    # 根据类型改变外观
    match type:
        PowerUpType.HEALTH:
            $Sprite2D.modulate = Color.GREEN
        PowerUpType.BOMB:
            $Sprite2D.modulate = Color.ORANGE
        PowerUpType.WEAPON_NORMAL:
            $Sprite2D.modulate = Color.WHITE
        # 其他类似

func _physics_process(delta):
    position.y += speed * delta
    if position.y > get_viewport_rect().size.y + 50:
        queue_free()

func _on_picked_up(other: Area2D):
    if other.get_parent().name == "Player":
        var player = other.get_parent()
        match type:
            PowerUpType.HEALTH:
                player.add_life(1)
            PowerUpType.BOMB:
                player.add_bomb(1)
            PowerUpType.WEAPON_NORMAL:
                player.set_weapon(player.WeaponType.NORMAL)
            PowerUpType.WEAPON_LASER:
                player.set_weapon(player.WeaponType.LASER)
            PowerUpType.WEAPON_SHOTGUN:
                player.set_weapon(player.WeaponType.SHOTGUN)
            PowerUpType.WEAPON_HEAVY:
                player.set_weapon(player.WeaponType.HEAVY)
        queue_free()
```

在 `player.gd` 中添加 `add_bomb` 方法：

```gdscript
func add_bomb(amount: int = 1):
    bomb_count = min(bomb_count + amount, max_bomb)
    bomb_changed.emit(bomb_count)
```

#### 2.5 敌机死亡时掉落道具

修改 `enemy.gd` 的 `destroy()` 方法，增加掉落概率：

```gdscript
func destroy():
    Global.add_score(score_value)
    # 掉落概率：10% 掉落道具
    if randf() < 0.1:
        drop_powerup()
    queue_free()

func drop_powerup():
    var powerup_scene = preload("res://scenes/powerup.tscn")
    var powerup = powerup_scene.instantiate()
    powerup.position = global_position
    # 随机选择道具类型
    var r = randi() % 100
    if r < 20:  # 20% 生命
        powerup.type = powerup.PowerUpType.HEALTH
    elif r < 40:  # 20% 炸弹
        powerup.type = powerup.PowerUpType.BOMB
    else:
        # 武器道具，随机四种武器
        var weapon_types = [powerup.PowerUpType.WEAPON_NORMAL, powerup.PowerUpType.WEAPON_LASER,
                            powerup.PowerUpType.WEAPON_SHOTGUN, powerup.PowerUpType.WEAPON_HEAVY]
        powerup.type = weapon_types[randi() % weapon_types.size()]
    get_tree().current_scene.add_child(powerup)
```

注意：需要将 `powerup.tscn` 中的 `Sprite2D` 纹理设置好（可以暂时用颜色区分，或者用不同形状的图片）。

### 第五步：平衡性调整建议

- 武器掉落概率：可以根据当前武器等级调整，低等级时更容易掉落同类型武器以促进升级。
- 分数奖励阈值：2^6=64k 可能较高，可降低为 2^5×1000=32000 或根据游戏节奏调整。
- 初始炸弹数量 3，最大 9 比较合理。
- 生命最大上限建议 5，防止过强。

## 本篇思考题

**1. 为什么我们要为每种武器单独存储等级，而不是一个全局武器等级？**  
> 答案：因为不同武器的性能差异很大，玩家可能根据情况切换武器，如果共享等级会导致切换后等级不合理（例如切换激光时继承了霰弹的高等级，破坏平衡）。独立等级允许玩家分别升级每种武器，增加策略性。

**2. 分数奖励生命函数 `check_score_reward` 中使用了 `last_reward_threshold` 变量。如果玩家在奖励生命后分数下降（例如扣分系统），会不会重复奖励？**  
> 答案：本游戏没有扣分机制，分数只会增加。所以不会重复奖励。如果有扣分，则需要更复杂的记录方式（例如记录已奖励过的阈值列表）。思考题引导读者考虑边界情况。

**3. 在 `drop_powerup()` 中，我们使用 `randf() < 0.1` 决定是否掉落。如果希望不同类型的敌机有不同的掉落概率（如精英敌机掉落率更高），应该如何修改？**  
> 答案：可以在敌机脚本中增加一个 `drop_chance` 变量，普通敌机0.1，精英敌机0.3。然后在 `destroy()` 中调用 `if randf() < drop_chance: drop_powerup()`。这样更灵活。
