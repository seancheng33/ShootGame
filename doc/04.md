## 第四篇：制作各关卡的内容并测试

本篇将实现完整的 8 个关卡流程。每个关卡包含：
- 多波次敌人生成（普通敌机、精英敌机）
- 关底 Boss 出现条件（消灭足够数量的敌机或时间触发）
- 过关后自动奖励一条生命
- 关卡之间的无缝切换（保留玩家分数、武器、炸弹、生命值）
- 测试所有关卡的可玩性

> 本篇假设前三篇已实现玩家、敌机、碰撞、分数系统（`Global` 自动加载）。我们还需要创建一个 **关卡管理器** 来控制波次生成。

---

### 第一步：设计关卡数据结构

为了便于调整和扩展，我们将每个关卡的配置存储为 **资源（Resource）** 或简单的字典。这里使用 `Dictionary` 定义在脚本中，方便快速修改。

在 `scripts` 文件夹下新建脚本 `level_data.gd`（作为自动加载或普通脚本？为了方便，我们直接在关卡管理器中定义）。

更规范的做法：创建一个 `LevelConfig` 资源类。但为了简化教程，我们在关卡管理器脚本中直接定义 8 个关卡的配置数组。

var current_level := 1

    # 为节省篇幅，这里只写出差异较大的第8关
    8: [
        {"type": EnemyType.NORMAL, "count": 20, "spawn_interval": 0.5, "start_delay": 0.0},
        {"type": EnemyType.ELITE, "count": 10, "spawn_interval": 1.0, "start_delay": 1.0},
        {"type": EnemyType.NORMAL, "count": 30, "spawn_interval": 0.4, "start_delay": 2.0},

# 每个关卡的 Boss 配置
var boss_config := {
    1: {"health": 20, "score_multiplier": 2000, "speed": 50, "shoot_cooldown": 1.0},
    2: {"health": 25, "score_multiplier": 4000, "speed": 55, "shoot_cooldown": 0.9},
    3: {"health": 30, "score_multiplier": 6000, "speed": 60, "shoot_cooldown": 0.8},
    4: {"health": 35, "score_multiplier": 8000, "speed": 65, "shoot_cooldown": 0.7},
    5: {"health": 40, "score_multiplier": 10000, "speed": 70, "shoot_cooldown": 0.6},

# 获取当前关卡的波次列表

    return level_waves.get(current_level, [])

func get_current_boss_config():
    return boss_config.get(current_level, boss_config[1])

# 切换到下一关

    if current_level < 8:

### 第三步：创建游戏主场景的关卡控制脚本

我们之前已有 `game.tscn`，现在需要重写它的脚本，使其能够根据 `LevelManager.current_level` 加载对应的波次和 Boss。

修改 `game.gd`（挂载在 `Game` 根节点上）：

```gdscript
extends Node2D

@onready var player = $Player
var enemy_scene = preload("res://scenes/enemy.tscn")
var elite_scene = preload("res://scenes/elite_enemy.tscn")
var boss_scene = preload("res://scenes/boss.tscn")

# 关卡生成相关
var current_wave_index := 0
var current_wave_data = null
var enemies_to_spawn := 0
var spawn_timer := 0.0
var wave_in_progress := false
var boss_spawned := false

# 用于延迟开始波次
var wave_start_delay := 0.0

func _ready():
    # 重置玩家状态（生命、武器、炸弹等已在 Player 脚本中保留，但新游戏需要重置）
    # 这里我们假设开始新游戏时已经重置了 Global 和 Player 相关数据
    # 简单起见，在进入 game 场景前应该重置分数等，但为了测试，我们在这里重置一下
    Global.score = 0
    # 开始第一关的波次生成
    start_level()

func start_level():
    current_wave_index = 0
    boss_spawned = false
    # 获取当前关卡的波次列表
    var waves = LevelManager.get_current_waves()
    if waves.size() == 0:
        # 如果没有配置波次，直接生成 Boss（用于测试）
        spawn_boss()
    else:
        start_next_wave()

func start_next_wave():
    if current_wave_index >= LevelManager.get_current_waves().size():
        # 所有波次结束，生成 Boss
        if not boss_spawned:
            spawn_boss()
        return
    current_wave_data = LevelManager.get_current_waves()[current_wave_index]
    enemies_to_spawn = current_wave_data["count"]
    spawn_timer = current_wave_data.get("spawn_interval", 1.0)
    wave_start_delay = current_wave_data.get("start_delay", 0.0)
    wave_in_progress = true
    current_wave_index += 1

func _process(delta):
    if wave_in_progress:
        if wave_start_delay > 0:
            wave_start_delay -= delta
            return
        
        if spawn_timer > 0:
            spawn_timer -= delta
        else:
            if enemies_to_spawn > 0:
                spawn_enemy_of_type(current_wave_data["type"])
                enemies_to_spawn -= 1
                spawn_timer = current_wave_data.get("spawn_interval", 1.0)
            else:
                # 当前波次完成，开始下一波
                wave_in_progress = false
                start_next_wave()

func spawn_enemy_of_type(type):
    var enemy
    match type:
        LevelManager.EnemyType.NORMAL:
            enemy = enemy_scene.instantiate()
        LevelManager.EnemyType.ELITE:
            enemy = elite_scene.instantiate()
    if enemy:
        enemy.position = Vector2(randf_range(50, get_viewport_rect().size.x - 50), -50)
        add_child(enemy)

func spawn_boss():
    boss_spawned = true
    var boss = boss_scene.instantiate()
    var cfg = LevelManager.get_current_boss_config()
    boss.health = cfg["health"]
    boss.speed = cfg["speed"]
    boss.score_value = LevelManager.current_level * 2000
    # 设置 Boss 的射击冷却等（可以在 boss.gd 中通过属性设置）
    boss.shoot_cooldown = cfg["shoot_cooldown"]
    boss.position = Vector2(get_viewport_rect().size.x / 2, -100)
    add_child(boss)
    # 监听 Boss 死亡信号，以便过关
    boss.tree_exited.connect(_on_boss_defeated)

func _on_boss_defeated():
    # 过关奖励一条命
    player.add_life(1)   # 需要在 player.gd 中实现 add_life 方法
    # 判断是否通关
    var has_next = LevelManager.next_level()
    if has_next:
        # 加载下一关：重新开始波次生成，保留玩家状态
        start_level()
    else:
        # 通关游戏，显示胜利界面（暂用打印）
        print("Game Completed!")
        get_tree().change_scene_to_file("res://scenes/win_screen.tscn")  # 需要创建胜利场景
```

### 第四步：为玩家添加 `add_life` 方法和生命值上限

在 `player.gd` 中添加：

```gdscript
func add_life(amount: int = 1):
    current_health = min(current_health + amount, max_health)
    health_changed.emit(current_health)
```

同时确保 `max_health` 初始为 3，并且生命值不会超过该值。

### 第六步：添加 Boss 死亡后切换关卡的逻辑

我们在 `boss.gd` 的 `destroy()` 中发出信号或直接调用 `queue_free()`，外部通过 `tree_exited` 连接。上面已经在 `game.gd` 的 `spawn_boss` 中连接了 `boss.tree_exited.connect(_on_boss_defeated)`。但需要确保 Boss 死亡时正确发出 `tree_exited`，这会在 `queue_free()` 时自动触发。

另外，需要处理一个细节：Boss 死亡后，如果关卡波次尚未完全结束（例如 Boss 提前生成），我们应该停止波次生成。当前设计是波次全部结束后才生成 Boss，所以没问题。

### 第八步：添加游戏启动时的重置逻辑

为了确保新游戏时所有状态重置（分数、生命、武器、炸弹、当前关卡为 1），我们需要在进入 `game.tscn` 前或在 `game.gd` 的 `_ready()` 中调用重置函数。

在 `global.gd` 中添加重置方法：

```gdscript
func reset_game():
    score = 0
    score_changed.emit(0)
```

在 `level_manager.gd` 中已有 `reset()` 方法重置关卡为 1。

然后在 `game.gd` 的 `_ready()` 中：

```gdscript
func _ready():
    Global.reset_game()
    LevelManager.reset()
    # 同时重置玩家内部状态，例如生命、武器、炸弹
    player.reset_state()  # 需要在 player.gd 中实现
    start_level()
```

在 `player.gd` 中添加：

```gdscript
func reset_state():
    current_health = max_health
    bomb_count = 3
    current_weapon = WeaponType.NORMAL
    weapon_level = 1
    shoot_cooldown = 0.2
    health_changed.emit(current_health)
    bomb_changed.emit(bomb_count)
    weapon_changed.emit(current_weapon)
    # 重置颜色为默认蓝色
    set_player_color(PlayerColor.BLUE)
```

## 本篇思考题

**1. 为什么将关卡配置放在自动加载的 `LevelManager` 中，而不是直接写在 `Game` 场景里？**  
> 答案：因为关卡配置需要在多个场景中访问（例如游戏主场景、UI 显示当前关卡、过关后切换关卡等）。使用自动加载的单例可以全局访问，便于维护和修改，也符合单一职责原则。

**2. 在 `start_next_wave()` 中，我们通过 `current_wave_index` 遍历波次列表。如果某波次敌人生成过程中玩家死亡导致场景重置，如何保证波次进度不丢失？**  
> 答案：当前设计下，场景重置会丢失进度。改进方案是将波次进度保存到 `LevelManager` 中（例如当前波次索引、剩余敌人数等），并在 `Game` 的 `_ready()` 中恢复。或者改用更健壮的存档系统。思考题希望读者意识到状态持久化的重要性。

**3. 在 Boss 生成前，如果玩家已经清空了所有波次，但 Boss 生成有延迟，此时玩家无事可做。有什么办法优化这种体验？**  
> 答案：可以在最后一波结束后立即生成 Boss，或者添加一个短暂的倒计时提示（例如“Boss 来袭”），也可以生成一些额外的小敌机填补空白。这种设计可以增加紧张感。
