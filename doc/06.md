## 第六篇：游戏的导出（Windows、macOS、Linux、HTML5）

本篇将指导您将完成的飞行射击游戏导出为可发布的桌面平台（Windows、macOS、Linux）以及网页版（HTML5）。我们会涵盖：
- 导出模板的下载与安装
- 为每个平台创建导出预设
- 调整导出选项（分辨率、图标、代码优化）
- HTML5 的特殊处理（服务器、跨域、全屏）
- 导出后的测试与常见问题解决

> 本教程适用于 Godot 4.6.2，确保您的游戏在编辑器中运行正常后再导出。

---

### 第一步：下载并安装导出模板

Godot 需要单独的导出模板文件才能将项目导出为可执行程序。

1. 打开 Godot 编辑器，点击顶部菜单 **编辑器 → 管理导出模板**。
2. 点击 **下载** 按钮，Godot 会自动匹配当前版本（4.6.2）的导出模板并下载。
   - 如果下载失败，可以手动从官网 https://godotengine.org/download/archive/ 下载对应版本的 `Godot_v4.6.2-stable_export_templates.tpz`。
   - 然后在管理窗口中点击 **安装从文件**，选择该文件。
3. 等待安装完成，关闭窗口。

> 注意：如果后续需要导出 Android/iOS 等平台，需要额外安装 SDK，但本游戏仅需要桌面和 Web，上述步骤已足够。

---

### 第二步：配置导出预设（Export Presets）

导出预设保存了导出配置（输出路径、图标、权限等）。我们需要为每个平台创建一个预设。

1. 点击菜单 **项目 → 导出**，打开导出面板。
2. 点击 **添加预设**，选择 **Windows Desktop**，命名为 `Windows`。
3. 同样添加 **macOS**、**Linux/X11**、**Web** 预设。

#### 2.1 Windows 预设配置

在 Windows 预设的 **选项** 选项卡中：

- **输出路径**：点击文件夹图标，选择 `build/windows/ShootGame.exe`（相对路径）。
- **图标**：可以准备一个 `.ico` 文件（建议 256x256），点击 `...` 选择。如果没有，留空使用默认图标。
- **代码优化**：
  - `Export Mode`：选择 `Release`（最终发布），`Debug` 用于测试。
  - `Texture Format`：保持 `s3tc`（通用）。
- **高级选项**：通常保持默认。

#### 2.2 macOS 预设配置

- **输出路径**：`build/macos/ShootGame.zip`（或直接 `.app` 目录）。
- **图标**：需要 `.icns` 文件（可用在线工具将 PNG 转换）。如果不提供，会使用 Godot 默认图标。
- **代码签名**：如果发布到 App Store 或需要避免网关警告，需配置签名证书。个人开发者可跳过，导出的 `.app` 可直接运行（用户会收到“未验证开发者”提示，可右键打开）。
- **注意**：在 macOS 上导出需要确保目标架构为 `x86_64` 和 `arm64`（Apple Silicon）。在 **导出** 面板的 `Options` → `Binary Format` 勾选 `Include ARM64` 和 `Include x86_64`，并勾选 `Create ZIP` 方便分发。

#### 2.3 Linux 预设配置

- **输出路径**：`build/linux/ShootGame.x86_64`（或任意名称）。
- **图标**：支持 `.png` 或 `.svg`。
- **注意**：Linux 导出文件没有扩展名，但通常加上 `.x86_64` 表示可执行文件。导出后需赋予执行权限 `chmod +x ShootGame.x86_64`。

#### 2.4 Web（HTML5）预设配置

这是最特殊的平台，需要特别设置：

- **输出路径**：`build/web/index.html`（Godot 会生成 HTML 文件和多个资源文件）。
- **选项**：
  - `Variant`：选择 `Regular`（标准），`Threads` 需要服务器支持跨域隔离，不推荐初学者。
  - `Texture Format`：选择 `WebP` 或 `s3tc`（Web 支持度好）。
  - `HTML5 导出选项`：
    - 勾选 `Custom HTML Shell`：可提供自定义的 `index.html` 模板（高级）。
    - 勾选 `Head Include`：可添加自定义 `<meta>` 标签或加载进度条样式。
  - **重要**：在 **导出** 面板下方，展开 **高级选项**，找到 `Export Type` 选择 `Release`，并确保 `Optimize for Size` 为开启。

> 注意：Web 导出生成的文件必须通过 HTTP 服务器访问，不能直接双击 HTML 文件（会有跨域限制）。

---

### 第三步：处理平台特定的代码差异

Godot 提供了 `OS.get_name()` 来判断当前运行平台。在游戏逻辑中，可能需要对 Web 版本做一些调整：

- **HTML5 全屏**：Web 上的全屏需要用户交互触发。可以在 UI 上添加一个“全屏”按钮，调用 `DisplayServer.window_set_mode(DisplayServer.WINDOW_MODE_FULLSCREEN)`。
- **输入设备**：Web 支持键盘鼠标，与桌面版相同，无需修改。
- **音频**：Web 需要在用户首次点击页面后才会启用音频。可以在启动场景加一个“点击开始”的覆盖层。
- **性能**：Web 版本的子弹数量过多可能导致掉帧，可适当减少屏幕上的敌机上限。

为了简单，本游戏不添加额外适配，直接导出也能运行。

---

### 第四步：导出并测试

#### 4.1 导出所有平台

在导出面板中，依次选择每个预设，点击 **导出项目**，等待 Godot 生成文件。

- Windows：生成 `.exe` 文件和 `.pck`（或单个 `.exe` 如果勾选了嵌入 PCK）。
- macOS：生成 `.app` 文件夹或 `.zip`。
- Linux：生成单个可执行文件和 `.pck`。
- Web：生成一个文件夹，内含 `index.html`、`.wasm`、`.pck` 等。

#### 4.2 测试桌面版本

- Windows：双击 `.exe` 运行。
- macOS：解压 `.zip`，将 `.app` 拖到应用程序文件夹，然后右键 → 打开（首次运行需允许）。
- Linux：在终端中给执行权限 `chmod +x ShootGame.x86_64`，然后 `./ShootGame.x86_64`。

确保游戏运行正常，音效、碰撞、关卡切换无报错。

#### 4.3 测试 Web 版本

由于安全限制，不能直接双击 `index.html`。你需要一个本地 Web 服务器。

**快速方法**（使用 Python 3）：
- 打开终端，进入 `build/web` 目录。
- 运行 `python -m http.server 8000`。
- 打开浏览器访问 `http://localhost:8000`。

**或者使用 Godot 内置 Web 服务器**：在导出面板 Web 预设旁边点击 **运行**，Godot 会自动启动临时服务器并打开浏览器。

测试内容：
- 游戏是否正常加载（进度条）。
- 键盘/鼠标输入是否响应（注意浏览器可能需要点击游戏区域以聚焦）。
- 全屏是否可用。
- 帧率是否流畅。

### 第五步：常见问题与解决方法

| 问题 | 可能原因 | 解决方案 |
|------|---------|----------|
| Windows 导出缺少 DLL 或运行报错 | 未包含运行时库 | 确保导出模板版本匹配。尝试在导出选项中勾选 `Embed Pck`。 |
| macOS 提示“已损坏” | 未签名 | 用户需执行 `sudo spctl --master-disable` 或右键打开。正式发布应签名。 |
| Web 版本无法加载 `.pck` 文件 | MIME 类型错误或跨域 | 服务器需正确配置 `.wasm` 和 `.pck` 的 MIME（`.wasm` → `application/wasm`）。本地测试用 http.server 会自动处理。 |
| Web 版本键盘无响应 | 浏览器未聚焦 | 添加一个“点击激活”覆盖层，或设置 `Window.onload` 时聚焦 canvas。 |
| 导出的游戏体积过大 | 包含调试符号或未压缩纹理 | 导出时选择 `Release` 模式，纹理压缩使用 `WebP`（Web）或 `s3tc`（桌面）。 |
| Linux 导出无法运行 | 缺少执行权限或库依赖 | 添加执行权限。如果依赖 SDL2，确保系统已安装。 |

---

### 第六步：发布与分发建议

- **Windows**：将 `.exe` 和 `.pck` 打包成 `.zip` 分发。如果希望单文件，可在导出选项中勾选 `Embed Pck`。
- **macOS**：导出为 `.zip`，用户解压后得到 `.app`。注意 zip 可能会破坏符号链接，建议使用 `dmg` 打包（第三方工具）。
- **Linux**：提供 `.tar.gz` 或 `.AppImage`（需要额外工具）。
- **Web**：将所有生成文件上传到任意静态网站托管服务（GitHub Pages、Netlify、itch.io 等）。注意添加 `index.html` 为入口。

## 本篇思考题

**1. 为什么导出的 Web 版本不能直接双击 `index.html` 运行？**  
> 答案：因为浏览器的同源策略（CORS）限制，从本地文件系统加载的 HTML 无法请求本地其他文件（如 `.wasm`、`.pck`）。必须通过 HTTP 服务器提供这些文件，浏览器才允许读取。

**2. 在导出 macOS 应用时，如果不进行代码签名，用户会遇到什么提示？如何解决？**  
> 答案：用户会看到“无法验证开发者”或“已损坏”的警告，无法直接打开。解决方法：右键点击应用，选择“打开”，然后在弹窗中确认打开。或者运行命令 `sudo spctl --master-disable` 关闭 Gatekeeper（不推荐）。正式发布应购买苹果开发者证书进行签名。

**3. 如果希望减少 Web 导出文件的体积，可以采取哪些措施？**  
> 答案：在导出预设中启用 `Optimize for Size`；使用纹理压缩格式（如 WebP）；将不需要的资源和脚本排除导出；启用 `Export PCK` 的压缩；使用 GDScript 而不是 C# 以减小 wasm 体积。
